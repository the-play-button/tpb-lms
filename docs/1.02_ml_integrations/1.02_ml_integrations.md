## Résultat : Pas de gain ML identifié ✅

**C'est le résultat attendu (99% des cas).**

---

## Évaluation des Features

| Feature | ML nécessaire ? | Solution retenue |
|---------|-----------------|------------------|
| Video progress tracking | Non | Calcul position/durée (code) |
| Quiz scoring | Non | Règles déterministes |
| XP calculation | Non | Arithmétique (50 video, 100 quiz) |
| Badge attribution | Non | Triggers conditionnels SQL |
| Leaderboard | Non | Vue SQL agrégée |
| Streak calculation | Non | Comptage jours consécutifs |

---

## Opportunités Évaluées et Rejetées

| Idée ML | Verdict | Motif du rejet |
|---------|---------|----------------|
| Recommandations de cours | ❌ | LMS interne, parcours imposés (pas de catalogue) |
| Transcription auto (Whisper) | ❌ | Vidéos courtes FR, ROI faible pour l'effort |
| Génération quiz LLM | ❌ | Non-déterministe, qualité variable, risque |
| Prédiction d'abandon | ❌ | Optimisation prématurée, pas de data historique |
| Classification contenu | ❌ | Metadata manuelle suffisante |

---

## Conclusion

Le code "à l'ancienne" couvre **100% des besoins** identifiés.

Pas de shiny objects ML. Pas de complexité inutile.
